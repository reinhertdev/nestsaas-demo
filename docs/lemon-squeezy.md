## Description (main)

Most SaaS boilerplates give you Next.js + API Routes and call it a "backend." NestSaaS gives you a real, production-grade NestJS backend — with proper modules, dependency injection, guards, and a clear separation between frontend and backend.

Built for indie developers and small teams who want to ship fast without cutting corners on architecture.

**What's included:**

- ✅ NestJS 11 backend (not just API Routes)
- ✅ Next.js 16 with App Router
- ✅ Supabase Auth — email/password, Google OAuth, Magic Link
- ✅ Stripe subscriptions — checkout, webhooks, customer portal
- ✅ Admin dashboard — user list, MRR tracking
- ✅ AWS ECS (Fargate) deployment config via CDK
- ✅ Docker Compose for local dev
- ✅ GitHub Actions CI/CD pipeline
- ✅ Prisma + PostgreSQL (Supabase)
- ✅ Claude API integration example
- ✅ Full TypeScript throughout

**Who is this for?**

- Developers who want a real backend architecture, not a monolithic Next.js app
- Teams planning to scale beyond a simple MVP
- Developers tired of spending weeks on auth, payments, and deployment setup every time they start a new project

**Why not ShipFast or Makerkit?**

Those are great if you're building on Vercel and don't need a separate backend. NestSaaS is for developers who want proper backend separation, enterprise-grade module structure, and the ability to deploy anywhere — including AWS ECS.
